Com funciona Internet?

Quin va ser l'origen d'Internet?

L'internet va sorgir d'un altre projecte anomenat ARPANET com a un projecte de recerca del departament de defensa. Paul Baran amb l'objectiu de construir un sistema de comunicacions que sobrevisqués un atac nuclear, va tenir la idea de trencar els missatges en blocs i enviar-los el més ràpid possible en totes direccions a través de la xarxa. I després, Vinton Cerf i Robert Khanvan crear el protocol en que es basa la comunicació a internet.

Avui dia, l'internet és una xarxa molt gran formada de moltes altres connexions independents que no té una central de control ni és controlada per ningú.

Capa física

Cables de coure (cables Ethernet)

El principal avantatge és el seu baix preu i el principal inconvenient és la pèrdua de senyal o interferència a uns pocs metres.

Fibra òptica

A la fibra òptica els bits es transmeten a través de la llum. El principal avantatge és que és un sistema de transmissió molt ràpid i que no té pèrdua de senyal a llargues distàncies. El principal inconvenient és que és un sistema molt fràgil i molt car.

Ones electromagnètiques

El principal avantatge és que és un sistema inalàmbric, i per tant mòbil. El principal inconvenient és que les ones no arriben molt lluny abans de que es distorsionin.

Wi-Fi

És una xarxa inalàmbrica que permet als dispositius connectarse entre sí a trevés d'ones de ràdio.

Capa d'enllaç de dades

Protocol ARP

Consisteix a enviar un missatge a tots els dispositius connectats a una xarxa local per preguntar quin ordenador té una determinada ip, per tant de que l'ordinador que busca li digui la seva adreça MAC.

Adreça MAC

La funció de l'adreça MAC és identificar un dispositiu. No hi ha cap dispositiu que tingui la mateixa adreça MAC que un altre.

Capa de xarxa

Un protocol és un conjunt de normes i estàndards que si tothom els utilitza permet la comunicació sense problemes.

IP (IPv4,IPv6)

És una adreça que identifica a cada dispositiu que forma part d'una xarxa. La utilitat d'aquest protocol és establir una connexió entre dispositius per tal de compartir adreces IP perquè els dispositius coneguin l'adreça IP de l'altre dispositiu per poder-se comunicar.

Diferència entre IPv4 i IPv6

La diferencia principal és que la versió IPv4 té 32 bits per adreça IP, en canvi, a la versió IPv6 les adreçes tenen 128 bits. Aquesta nova versió permet identificar a molts més dispositius, específicament 340 trillions a diferència del IPv4 que només permet identificar a 4 mil milions de dispositius.

Servidor DHCP (Protocol dinàmic de configuració de host)

Té la funció d'assignar una adreça IP a els dispositius que formen part d'una xarxa y d'administrar-les sense necessitat de configuració manual. Aquest seervidor normalment es troba als routers domèstics.
Si no s'utilitzés un servidor d'aquest tipus tots els dispositius s'haurien de configurar manualment.

Capa de transport

Què és un router?

Un router és un ordinador especial que controla el trànsit de dades a través de la xarxa.
A més amb un router domèstic podem crear una xarxa local o LAN en la qual uns quants dispositius es connecten per cable o de forma inalámbrica i també es pot utilitzar com a switch (commutador).

TCP

Comprova que totes les dades enviades com paquets hagin arribat a l'ordinador i els ordena, com un servei de correu amb garantia.
El protocol IP ens permet conèixer l'adreça IP de l'ordinador amb el qual ens volem comunicar, si no utilitzéssim aquest protocol no podríem enviar ni rebre dades, per tant, tampoc podríem usar el protocol TCP.
El TCP revisa tots els paquets que s'han rebut i si n'hi falta algun, demana que els paquets que no han arribat es tornin a enviar. Si no comptéssim amb aquest protocol hi hauria molts errors, ja que aquest protocol comprova que totes les dades arribin al seu destí, sense ell podrien faltar dades.

UDP

El protocol UDP transfereix dades entre dos ordinadors, però aquest protocol no comprova ni l'ordre ni s'hi han arribat al destí tots els paquets. Un dels casos on es fa servir és per a la reproducció de vídeos.

Capa d'aplicació

Protocol DNS

La seva funció és associar un nom de domini a una IP. Quan un ordinador sol·licita accedir a un domini, el servidor DNS busca la IP associada amb el nom de domini, la guarda i respon a la sol·licitud.
Molts hakers modifiquen l'IP associada a un nom de domini per una d'incorrecta. Aixó ho fan per fer-se passar per la pàgina real, enganyen a els usuaris fent que acabin en una web falsa. Per aquesta raó, existeixen els certificats digitals que són semblants a un document d'identitat oficial que assegura que el lloc web és realment el lloc web que diu que és.

HTTP i HTML

URL

Un URL o Uniform Resource Locator és la direcció que s'utilitza per accedir a un lloc web. I per veure aquestes webs utilitzem navegadors webs.

HTML

El llenguatge HTML és el codi que s'utilitza per presentar pàgines web, és a dir, com estan estructurades o que contenen.

HTTP i HTTPS

El protocol HTTP és un protocol que es fa servir per definir el format i com es transmeten els missatges en la comunicació de navegadors web i servidor. El protocol HTTP comunica un ordinador amb un altre ordinador anomenat servidor.

La diferència entre l'HTTP i l'HTTPS és, que l'HTTPS indica que la connexió es segura i que no es poden veure les dades que s'envien, en canvi l'HTTP no.


Petició GET: serveix per demanar informació a un servidor.
No, s'envien per separat ja que aquestes tenen una URL pròpia. Per invocarles, el navegador envia peticions GET per a cada imatge y la pàgina les mostra segons van arribant.

Petició POST: serveix per enviar dades a un servidor.

Cookies: serveixen perquè una pàgina web recordi les teves dades, per poder fer-les servir més endavant.
